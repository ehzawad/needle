File-defined-scope bot — 50-scenario evaluation

Base Qwen3-4B-Instruct (4-bit) + a per-query LLM scope-gate over a 16-card scope file. Live decisions on an RTX A5000.

This model is NOT fine-tuned / not SFT-trained. There is zero training here. It is the stock instruct model reading the scope file (seed16/cards.json) on every query. The gate decides ANSWER / CLARIFY / ABSTAIN from the query + the cards' included/excluded lists; grounded answers come only from the selected card's approved facts. Edit the file → the domain changes, no retraining. (The earlier QLoRA SFT plan was abandoned per a Codex Council review.)

How the over/under-clarify was fixed

A deterministic scope policy, not a smarter prompt. The earlier gate over-clarified clear in-scope queries and guessed on ambiguous ones. Prompt-tuning alone was whack-a-mole (fixed two cases, regressed a third) because the base model won't reliably obey a prompt rule — in a bank context "PIN" defaults to "card PIN". The fix: 6 collision-prone cards declare machine-checkable required_discriminators (in-scope vs out-of-scope cues); the LLM only picks a card, then scope_policy.py enforces the decision fail-closed, downgrade-only (ANSWER → CLARIFY/ABSTAIN, never a promotion). Bare “change my PIN” → CLARIFY; “change my SIM PIN” → ABSTAIN (which also hardens the 0-leak property); cards with no collision print REQUIRES: none. This eval is the base gate + policy with no exemplar bank.
Real conversations train the recall layer. Static cues miss novel phrasings (the safe direction — they over-clarify). Production chat closes that gap for free: the next user turn labels the prior gate decision (a correction after an ANSWER ⇒ should have clarified; a “thanks” ⇒ the card was right; silence ⇒ no signal). Mined, human-reviewed labels become a KB-versioned exemplar bank injected into the gate — no retraining. See FEEDBACK_PIPELINE.md.

Honest read

A dev gate, not a certificate. 0/25 leaks, 20/20 right-card in-scope, 5/5 ambiguous→clarify is on 50 hand-written scenarios. 0/25 has a one-sided 95% upper bound near 11%; a <1% leak claim needs zero failures across ≥299 labeled cases on a named distribution (Clopper–Pearson). Identical-text doppelgängers remain information-theoretically inseparable — the policy asks instead of guessing, it does not read minds.
